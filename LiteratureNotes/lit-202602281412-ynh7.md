# LiveKit Basics Overview

参照元: [[SourceNotes/LiveKit Agents Documentation.md|LiveKit Agents Documentation]]
ロードマップ: [[StructureNotes/LiveKit Agent Framework 学習ロードマップ.md|学習ロードマップ]]

## What（何についてか）

LiveKitの全体像をつかむための入門ページ。プラットフォームが何者で、何を提供しているかを俯瞰する。

## Why（なぜ必要か）

AI-native なリアルタイムアプリ・音声エージェントを作るためのインフラとして LiveKit がある。
コア概念を理解しておかないと、Agents Framework の学習で「なぜそうなっているのか」が見えなくなる。

## How（どう動くか）

LiveKit のアーキテクチャは **Rooms / Participants / Tracks** の3層で構成されている。

- **Room**: 通信が起きる仮想空間
- **Participant**: Room に参加するエンティティ（人間もAIエージェントも同じ扱い）
- **Track**: Participant 間を流れるメディアストリーム（音声・映像・データ）

AI エージェントも「一人の Participant」として Room に参加し、他の Participant と同じインフラ上で動く。これが LiveKit の設計思想の核心。

## Key Concepts

| 用語 | 説明 |
|---|---|
| LiveKit CLI | プロジェクト管理・テンプレート初期化・デプロイ用のコマンドラインツール |
| LiveKit Cloud | フルマネージドのグローバル分散プラットフォーム。エージェントホスティング・推論・テレフォニー・Observability を統合提供 |
| Access Token | LiveKit サーバーへの接続に使う認証トークン |
| WebRTC | LiveKit が採用する接続プロトコル |
| Room | 通信が行われる仮想空間。全てのやりとりはここで起きる |
| Participant | Room に参加するエンティティ（ユーザー・エージェント両方） |
| Track | Participant 間を流れるメディアストリーム（音声・映像・データ） |
| Agents Framework | LiveKit 上でAIエージェントを構築するためのフレームワーク |

## 一言まとめ

LiveKit は「Room という仮想空間に Participant が集まって Track を交換する」プラットフォームで、AIエージェントも人間と同じ Participant として扱われる。
