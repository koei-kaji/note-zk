# 直接入力ソース

## 1. 思考内容の確認

引数が提供された場合は`$ARGUMENTS`を解析します。
提供されていない場合は、ユーザーと対話して思考内容を確認します。

### 引数解析

`$ARGUMENTS`から以下を抽出：
- 第1引数: タイトル（オプション）
- 第2引数以降: 思考内容

例：
```
/flt "アイデアのタイトル" "具体的な思考内容や気づき"
```

### 対話による確認

引数が不足している場合、以下を確認：
- どのような思考やアイデアですか？
- 背景や文脈はありますか？
- 関連する概念やトピックはありますか？

## 2. メインプロセスへ

思考内容が確認できたら、メインプロセス（@.claude/commands/flt.md）の「6段階対話プロトコル」に進みます。
